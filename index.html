Types of Transactions

  • Batch Processing - Transactions are processed in bulk.
  • Real-time Processing - Transactions are processed instantly.
  • Based on User Involvemen - Only one user can interact with the database at a time.
  • Based on Data Operation - Designed for complex queries and data analysis
  • Based on Application - Used in various applications, like MySQL and PostgreSQL.

ACID Properties

  • Transactions follow the ACID properties: Atomicity, Consistency, Isolation, and Durability.


  • Atomicity – Ensures that a transaction is treated as a single unit, meaning it either completes fully or not at all. If any part fails, the entire transaction is rolled back.
  • Consistency – Guarantees that the database transitions from one valid state to another. Any transaction must follow defined constraints, ensuring data integrity.
  • Isolation – Ensures that transactions are executed independently, preventing interference. Even if multiple transactions run concurrently, their results should be as if they were executed sequentially.
  • Durability – Once a transaction is committed, its changes are permanently saved, even in the event of system failures.
